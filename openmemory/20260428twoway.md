# 20260428twoway.md — 酒店客服双向对话业务逻辑

## 概述

本文档定义了 OpenMemory 酒店客服场景的**双向对话（Two-Way Conversation）**业务逻辑。
系统需支持：
1. **客服视角**：从历史记忆中检索客户偏好/历史记录，给出精准回复
2. **客户视角**：客户发出咨询，系统将关键信息写入记忆

---

## 业务场景分类

| 场景 | 触发关键词 | 系统行为 |
|------|-----------|---------|
| 预订咨询 | 预订、订房、空房、入住时间 | 查询记忆中客户历史偏好（楼层/床型/无烟房）并推荐 |
| 入住办理 | check-in、入住、押金、房卡 | 提取上次入住记录，自动填充常见需求 |
| 设施询问 | 游泳池、健身房、早餐、WIFI | 匹配设施描述，记录客户感兴趣的设施 |
| 投诉处理 | 投诉、噪音、脏、不满意、退款 | 记录投诉内容，触发优先级升级标记 |
| 退房结算 | 退房、check-out、账单、发票 | 检索本次消费记录，生成摘要 |

---

## 对话流程（双向）

```
客户消息 ──▶ [OpenMemory API]
                │
                ├── 1. search_memories(query, user_id)  ← 检索历史偏好
                │         ↓
                │   相关记忆列表
                │
                ├── 2. 构建上下文 context = {customer_history + new_query}
                │
                ├── 3. LLM生成回复（含个性化内容）
                │         ↓
                │   客服回复文本
                │
                └── 4. add_memory(dialog_summary, user_id)  ← 写入本次关键信息
                            ↓
                        记忆持久化 (FAISS / vector store)
```

## API 调用规范

### 写入记忆（客户说话时）

```python
POST /api/v1/memories/
{
  "user_id": "wechat_{openid}",
  "text": "{客户说的话}",
  "metadata": {
    "source": "wechat",
    "type": "hotel_consultation",
    "scene": "booking|checkin|facilities|complaint|checkout"
  }
}
```

### 检索记忆（客服回复前）

```python
GET /api/v1/memories/?user_id=wechat_{openid}&search_query={关键词}
```

### 场景识别规则

```python
SCENE_RULES = {
    "booking":     ["预订", "订房", "空房", "入住时间", "几号", "价格"],
    "checkin":     ["入住", "check-in", "押金", "房卡", "升级"],
    "facilities":  ["早餐", "游泳", "健身", "WIFI", "停车", "商务中心"],
    "complaint":   ["投诉", "噪音", "脏", "不满", "退款", "赔偿"],
    "checkout":    ["退房", "check-out", "账单", "发票", "报销"],
}
```

## 记忆提取规则

mem0 会自动从对话中提取以下关键事实：

| 类型 | 示例提取内容 |
|------|-------------|
| 客户偏好 | "喜欢高楼层" / "需要无烟房" / "大床房" |
| 历史行为 | "上次入住302，满意" / "经常预订商务套房" |
| 投诉记录 | "2024年12月投诉空调噪音" |
| 设施偏好 | "每天使用健身房" / "需要接送机服务" |
| 联系信息 | "公司报销，需要增值税发票" |

## Mock 模式 vs 生产模式

| 条件 | 模式 | 行为 |
|------|------|------|
| 无 API Key（`MOCK_MODE=true`） | Mock | 记忆写入 SQLite + FAISS；分类用关键词规则；不调用 LLM |
| 有 QWEN_API_KEY 或 DASHSCOPE_API_KEY | 生产 | 调用 Qwen API 提取记忆；调用 Qwen API 生成回复 |
| 有 OPENAI_API_KEY | 生产 | 调用 OpenAI API |

### 降级策略（Graceful Degradation）

```
LLM 调用失败
    ↓
[自动降级] 使用关键词规则分类 (categorization.py: _mock_categorize)
    ↓
[记忆存储] 仍然写入 SQLite 数据库 (content 字段)
    ↓
[向量索引] FAISS 更新（如 embedder 也失败则跳过向量化）
    ↓
[返回] 200 OK 带 warning 字段
```
