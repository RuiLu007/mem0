## 一、背景与目标

OpenMemory 项目原本只支持 OpenAI 作为大模型提供商。用户需要接入阿里云通义千问（DashScope），同时要求架构可扩展，方便后续接入更多 Provider（Ollama、DeepSeek、Anthropic 等）。

Qwen 的调用方式与 OpenAI SDK 完全兼容，只需更换 `base_url` 和 `api_key`：

```python
from openai import OpenAI
client = OpenAI(
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)
```

## 二、架构设计思路

### 2.1 问题分析

原代码的问题：
- `memory.py`：`_build_openai_llm_config` 硬编码 OpenAI API key env var；默认 embedder 永远用 openai
- `categorization.py`：`OpenAI(api_key=api_key)` 硬编码，model 硬编码 `gpt-4o-mini`
- `config.py`：`LLMConfig` 没有 `openai_base_url` 字段，无法通过 API 配置 Qwen 端点

### 2.2 核心设计：Provider 注册表

创建 `api/app/utils/providers.py`，采用**数据驱动注册表模式**（Registry Pattern）：

```
用户配置 (env var LLM_PROVIDER=qwen)
    ↓
providers.PROVIDER_REGISTRY["qwen"]
    ↓
ProviderDef(
    mem0_provider="openai",   # mem0 内部仍用 openai 驱动
    api_key_env="DASHSCOPE_API_KEY",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    default_llm_model="qwen-plus",
    default_embedder_model="text-embedding-v3",
)
    ↓
memory.py 生成 mem0 config:
{
  "provider": "openai",
  "config": {
    "model": "qwen-plus",
    "api_key": "<DASHSCOPE_API_KEY>",
    "openai_base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1"
  }
}
```

**关键洞察**：mem0 的 OpenAI provider 支持 `openai_base_url` 参数，因此所有 OpenAI-compatible 的服务（Qwen、DeepSeek、Azure OpenAI 等）都能复用同一个驱动，只需注册不同的 base_url。

### 2.3 DB 配置规范化

当用户通过前端 UI 将 provider 设为 `"qwen"` 并存入数据库时，`get_memory_client()` 读取 DB 配置后，需要在传给 mem0 之前做翻译：

```
DB: {"provider": "qwen", "config": {...}}
     ↓ _normalize_provider_section()
mem0: {"provider": "openai", "config": {"openai_base_url": "...", "api_key": "env:DASHSCOPE_API_KEY", ...}}
```

## 四、providers.py 核心数据结构

```python
@dataclass
class ProviderDef:
    mem0_provider: str           # mem0 内部驱动名
    api_key_env: str             # API key 环境变量名
    default_llm_model: str
    default_embedder_model: str
    base_url: Optional[str] = None   # OpenAI-compatible 端点

PROVIDER_REGISTRY = {
    "openai": ProviderDef(mem0_provider="openai", api_key_env="OPENAI_API_KEY", ...),
    "qwen":   ProviderDef(mem0_provider="openai", api_key_env="DASHSCOPE_API_KEY",
                          base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
                          default_llm_model="qwen-plus",
                          default_embedder_model="text-embedding-v3"),
    "ollama": ProviderDef(mem0_provider="ollama", api_key_env="", ...),
    # 扩展只需加一行 ↑
}
```

    default_embedder_model="",
