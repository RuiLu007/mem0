# log20260428_start.md — OpenMemory 全流程部署与运行手册

> 版本：2026-04-28  
> 架构：FastAPI (后端) + Next.js 15 (前端) + FAISS (向量存储) + SQLite (关系数据库)

---

## 一、系统架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户/浏览器                              │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP  :3000
┌────────────────────────────▼────────────────────────────────────┐
│              Next.js 15 前端  (ui/)                              │
│  页面：/        memories 列表                                    │
│        /chat    酒店咨询聊天界面                                  │
│        /settings  LLM/Provider 配置                             │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP  :8765
┌────────────────────────────▼────────────────────────────────────┐
│              FastAPI 后端  (api/)                                │
│                                                                  │
│  路由模块：                                                       │
│  /api/v1/memories   — CRUD 记忆条目                              │
│  /api/v1/apps       — App 管理                                   │
│  /api/v1/config     — LLM/向量库配置（含 /providers）             │
│  /api/v1/chat/      — 酒店聊天接口（关键词匹配 + 记忆检索）         │
│  /mcp               — MCP 协议接口                               │
│                                                                  │
│  核心组件：                                                       │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────────────────┐  │
│  │  mem0 Memory │  │  SQLite DB   │  │ Provider Registry     │  │
│  │  (FAISS后端) │  │  记忆元数据   │  │ openai/qwen/ollama... │  │
│  └──────┬───────┘  └──────────────┘  └───────────────────────┘  │
│         │                                                        │
└─────────┼────────────────────────────────────────────────────────┘
          │ 文件读写
┌─────────▼────────────────────────────────────────────────────────┐
│              FAISS 本地索引  (api/faiss_store/)                   │
│  index.faiss  — 向量索引二进制                                    │
│  index.pkl    — id→metadata 映射                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 各模块通信方式

| 连接 | 协议 | 说明 |
|------|------|------|
| 浏览器 → 前端 | HTTP :3000 | Next.js App Router SSR/CSR |
| 前端 → 后端 | HTTP :8765 | REST API (NEXT_PUBLIC_API_URL) |
| 后端 → FAISS | 文件 I/O | mem0 直接读写 faiss_store/ 目录 |
| 后端 → SQLite | SQLAlchemy | openmemory.db 本地文件 |
| 后端 → LLM | HTTPS | OpenAI/Qwen/Ollama API |

### 2.2 Python 依赖安装

```bash
cd /root/01userprofile_ai/mem0/openmemory/api

pip install fastapi uvicorn sqlalchemy mem0ai mcp \
    faiss-cpu sentence-transformers tenacity \
    python-dotenv pydantic fastapi-pagination \
    --break-system-packages
```

> 注意：Debian/Ubuntu 系统 Python 可能需要 `--break-system-packages`

### 2.3 Node.js 依赖安装

```bash
cd /root/01userprofile_ai/mem0/openmemory/ui
npm install --legacy-peer-deps
```

## 三、配置文件

### 3.1 后端环境变量  `api/.env`

```env
# ── 基础配置 ─────────────────────────────
DATABASE_URL=sqlite:///./openmemory.db
USER=hotel_guest

# ── 向量数据库：FAISS（本地，零外部依赖）────
FAISS_PATH=./faiss_store
# FAISS_DIMS=1536            # 可选，默认 1536
# FAISS_DISTANCE=cosine      # 可选：cosine|l2|ip

# ── Mock 模式（无 API Key 时使用关键词规则）─
MOCK_MODE=true
OPENAI_API_KEY=sk-mock-no-key-needed

# ── 使用真实 Qwen 时取消下面注释 ──────────
# MOCK_MODE=false
# LLM_PROVIDER=qwen
# DASHSCOPE_API_KEY=sk-你的密钥
# LLM_MODEL=qwen-plus
# EMBEDDER_MODEL=text-embedding-v3

# ── 使用真实 OpenAI 时取消下面注释 ─────────
# MOCK_MODE=false
# LLM_PROVIDER=openai
# OPENAI_API_KEY=sk-你的OpenAI密钥
```

### 3.2 前端环境变量  `ui/.env.local`

```env
NEXT_PUBLIC_API_URL=http://localhost:8765
NEXT_PUBLIC_USER_ID=hotel_guest
```

## 四、服务启动步骤

### Step 1：启动后端 API

```bash
cd /root/01userprofile_ai/mem0/openmemory/api

# 前台运行（看日志）
uvicorn main:app --host 0.0.0.0 --port 8765 --reload

# 或后台运行
nohup uvicorn main:app --host 0.0.0.0 --port 8765 \
    > ../api_server.log 2>&1 &
echo "后端 PID: $!"
```

**预期启动日志**：
```
Auto-detected vector store: faiss with config: {...}
Auto-detected LLM provider: openai (mem0: openai)
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8765
```

### Step 2：（可选）初始化酒店演示数据

```bash
cd /root/01userprofile_ai/mem0/openmemory/api
MOCK_MODE=true python seed_hotel_data.py
```

**预期输出**：
```
✅ 成功植入 48 条酒店咨询记忆
```

### Step 3：启动前端

```bash
cd /root/01userprofile_ai/mem0/openmemory/ui

# 前台开发模式
npm run dev

# 或后台
nohup npm run dev > ../ui_server.log 2>&1 &
```

**预期输出**：
```
▲ Next.js 15.x
- Local:        http://localhost:3000
- Network:      http://0.0.0.0:3000
✓ Ready in 2.3s
```

## 六、切换向量数据库（无需改代码）

```bash
# 切换到 Qdrant（需先启动 Qdrant 服务）
docker run -d -p 6333:6333 qdrant/qdrant
# 在 .env 添加：
QDRANT_HOST=localhost
QDRANT_PORT=6333
# 删除或注释 FAISS_PATH

# 切换到 Milvus
docker run -d -p 19530:19530 milvusdb/milvus:latest
# 在 .env 添加：
MILVUS_HOST=localhost
MILVUS_PORT=19530
# 删除或注释 FAISS_PATH

# 强制指定（覆盖自动检测）
VECTOR_STORE=faiss   # 或 qdrant / milvus / chroma / ...
```

## 七、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `faiss-cpu not found` | 未安装依赖 | `pip install faiss-cpu --break-system-packages` |
| 前端空白页 | 后端未启动 | 先启动后端 Step 1 |
| 记忆查询为空 | 未初始化数据 | 执行 Step 2 |
| LLM 调用失败 | 无 API Key | 保持 `MOCK_MODE=true` |
| Port 8765 占用 | 旧进程 | `kill $(lsof -t -i:8765)` |
