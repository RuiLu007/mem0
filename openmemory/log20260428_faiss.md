# log20260428_faiss.md — FAISS 向量数据库迁移日志

> 记录时间：2026-04-28  
> 项目：OpenMemory (`openmemory/`)  
> 任务：将向量数据库从 Qdrant 替换为 FAISS，采用策略模式设计，确保可平滑迁移至 Milvus 等其他后端

---

## 一、背景与目标

OpenMemory 原始代码默认使用 Qdrant 作为向量数据库（需独立部署容器），对于轻量本地运行场景过重。

目标：
1. 以 **FAISS** 作为主要（默认）向量存储，零外部依赖，纯本地文件持久化
2. 代码采用**策略模式（Strategy Pattern）**，通过抽象基类统一接口，支持平滑迁移
3. 为未来接入 **Milvus** 等生产级向量数据库预留干净扩展点

---

## 二、架构设计思路

### 2.1 原代码问题

`memory.py` 的 `get_default_memory_config()` 中存在一个 90 行的 `if/elif` 链，每次新增一种向量数据库就要直接在函数体里增加分支，职责不单一，难以维护：

```python
# 原代码（问题示例）
if os.environ.get('CHROMA_HOST') and os.environ.get('CHROMA_PORT'):
    vector_store_provider = "chroma"
    vector_store_config = {...}
elif os.environ.get('QDRANT_HOST') and os.environ.get('QDRANT_PORT'):
    vector_store_provider = "qdrant"
    vector_store_config = {...}
# ... 8 个 elif 分支 ...
elif os.environ.get('FAISS_PATH'):
    vector_store_provider = "faiss"
    vector_store_config = {...}
else:
    vector_store_provider = "qdrant"  # 默认 Qdrant，需要外部服务
```

### 2.2 策略模式设计

```
                    ┌─────────────────────┐
                    │   VectorStoreBase   │  ← 抽象基类 (ABC)
                    │  (Strategy 接口)     │
                    │  + provider_name    │
                    │  + get_mem0_config()│
                    └──────────┬──────────┘
                               │ 继承
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
┌───────▼──────┐   ┌───────────▼──────┐   ┌──────────▼──────┐
│FaissVector   │   │QdrantVector      │   │MilvusVector      │
│Store         │   │Store             │   │Store             │
│(默认/主用)    │   │(兼容保留)         │   │(未来迁移目标)     │
└──────────────┘   └──────────────────┘   └─────────────────┘
        + Chroma, pgvector, Redis, Weaviate, Elasticsearch, OpenSearch

                    ┌─────────────────────┐
                    │  detect_vector_store│  ← 工厂函数
                    │  () -> VectorStore  │    基于环境变量自动选择
                    │  Base               │    默认返回 FaissVectorStore
                    └─────────────────────┘
```

### 2.3 FAISS 持久化机制

FAISS 本身是内存索引，持久化通过 mem0 库实现：

```
{FAISS_PATH}/
├── index.faiss    ← 二进制 FAISS 索引（向量数据）
└── index.pkl      ← Python pickle（id → metadata 映射）
```

- **首次启动**：mem0 自动创建目录和索引文件
- **重启后**：mem0 从磁盘 load 已有索引，数据不丢失
- **`FaissVectorStore.get_mem0_config()`** 中调用 `os.makedirs(path, exist_ok=True)` 确保目录预先存在

### 2.4 迁移到 Milvus 的路径

当需要生产化部署时，迁移 Milvus 只需：
1. 设置环境变量 `MILVUS_HOST=xxx MILVUS_PORT=19530`
2. `detect_vector_store()` 自动选择 `MilvusVectorStore`，无需改代码
3. 或强制指定：`VECTOR_STORE=milvus`

## 四、vector_stores.py 代码结构

```python
class VectorStoreBase(ABC):
    COLLECTION = "openmemory"
    DEFAULT_DIMS = 1536
    
    @property
    @abstractmethod
    def provider_name(self) -> str: ...
    
    @abstractmethod
    def get_mem0_config(self) -> dict: ...


class FaissVectorStore(VectorStoreBase):
    """
    环境变量：
      FAISS_PATH     持久化目录（默认 ./faiss_store）
      FAISS_DIMS     向量维度（默认 1536）
      FAISS_DISTANCE 距离策略：cosine|l2|ip（默认 cosine）
    """
    def get_mem0_config(self) -> dict:
        os.makedirs(self.path, exist_ok=True)  # 确保目录存在
        return {
            "provider": "faiss",
            "config": {
                "collection_name": self.collection_name,
                "path": self.path,
                "embedding_model_dims": self.embedding_dims,
                "distance_strategy": self.distance_strategy,
            }
        }


class MilvusVectorStore(VectorStoreBase):
    """
    环境变量：
      MILVUS_HOST / MILVUS_PORT / MILVUS_TOKEN / MILVUS_DB_NAME
    """
    def get_mem0_config(self) -> dict:
        return {
            "provider": "milvus",
            "config": {
                "url": f"http://{self.host}:{self.port}",
                "token": self.token,
                "embedding_model_dims": self.embedding_dims,
                "metric_type": "COSINE",
                ...
            }
        }


def detect_vector_store() -> VectorStoreBase:
    """
    优先级：Chroma > Qdrant > Weaviate > Redis > pgvector >
            Milvus > Elasticsearch > OpenSearch > FAISS（默认）
    可通过 VECTOR_STORE=faiss|qdrant|milvus|... 强制指定
    """
    if os.environ.get("CHROMA_HOST"): return ChromaVectorStore()
    if os.environ.get("QDRANT_HOST"): return QdrantVectorStore()
    # ...
    return FaissVectorStore()   # 默认
```

## 六、FAISS vs 其他向量库对比

| 特性 | FAISS | Qdrant | Milvus |
|------|-------|--------|--------|
| 部署方式 | 本地文件 | Docker 容器 | Docker/K8s |
| 外部依赖 | 无 | 需要服务 | 需要服务 |
| 持久化 | 文件系统 | 本地/云 | 本地/云 |
| 适合场景 | 开发/小规模 | 中等规模 | 大规模生产 |
| 切换成本 | — | 改 env var | 改 env var |

## 七、从 FAISS 迁移到 Milvus

```bash
# 1. 启动 Milvus 服务（已有 Docker）
docker run -d --name milvus -p 19530:19530 milvusdb/milvus:latest

# 2. 修改 .env
MILVUS_HOST=localhost
MILVUS_PORT=19530
# 删除或注释 FAISS_PATH

# 3. 重启 API（detect_vector_store() 自动检测到 Milvus）
# 无需改任何代码
```
